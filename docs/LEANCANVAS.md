## 1. 문제

> 해결하려는 핵심 고통점

- **분산 관리의 비효율**: 모임 주최자가 공지는 카카오톡, 참여자 명단은 스프레드시트, 카풀은 개인 DM, 정산은 별도 앱으로 나눠 처리한다.
- **참여 진입 장벽**: 기존 도구는 로그인·앱 설치가 필요해 참여자의 즉각적인 신청이 어렵다.
- **정산 수작업**: 1/n 계산과 납부 현황 추적을 직접 해야 하며 오류가 잦다.
- **주최자 과부하**: 참여자 수락·거절·공지·카풀 조율을 여러 채널에서 동시에 처리해야 한다.

### 기존 대안

| 대안              | 한계                                              |
| ----------------- | ------------------------------------------------- |
| 카카오톡 단체방   | 참여자 명단·정산 관리 불가                        |
| 네이버 밴드       | 비로그인 참여 불가, 정산 미지원                   |
| 구글 스프레드시트 | 실시간 업데이트 불편, 링크 공유 시 정보 노출 우려 |
| 정산 앱 (토스 등) | 모임 공지·참여자 관리 기능 없음                   |

## 2. 고객 세그먼트

> 누구의 문제를 해결하는가

### 얼리 어답터 (집중 타겟)

- **소규모 모임 주최자** (10~30명): 수영·헬스·친구 정기 모임을 반복적으로 운영하는 사람
  - 매달 1회 이상 모임을 기획하고 공지·참여자 관리를 담당
  - 카카오톡으로 수동 관리 중, 반복 작업에 피로감 큼

### 보조 세그먼트

- **모임 참여자** (로그인 사용자 + 비로그인 게스트)
  - 링크 하나로 참여 신청만 하면 되는 낮은 진입 장벽 요구
  - 카풀·정산 현황을 앱 안에서 확인하고 싶은 수요

## 3. 가치 제안

> 왜 우리 앱인가 — 고유한 한 줄 메시지

### 핵심 메시지

> **링크 하나로 모임 공지·참여·카풀·정산을 모두 처리한다.**

### 고유 가치 제안 (UVP)

- **주최자**: 흩어진 도구 대신 하나의 대시보드에서 이벤트 전 과정을 관리
- **참여자**: 앱 설치·회원가입 없이 링크 클릭 → 이름 입력만으로 참여 신청 완료
- **정산**: 비용 등록 시 1/n이 자동 계산되고, 납부 현황이 실시간으로 추적됨

### 상위 개념

> 소규모 모임을 위한 **올인원 운영 도구**

## 4. 솔루션

> 문제를 해결하는 핵심 기능 (최소한으로)

### Phase 1 — MVP (2주)

- **`join_code` 공개 링크**: UUID 기반 참여 신청 URL 자동 생성 → 카카오톡에 붙여넣기만 하면 됨
- **비로그인 참여 신청**: 이름·이메일 입력만으로 신청 가능 (Supabase anon RLS 활용)
- **주최자 대시보드**: 참여자 수락·거절, 공지 작성·핀 고정을 한 화면에서 처리
- **이벤트 CRUD**: 카테고리(수영/헬스/친구/기타), 날짜, 장소, 최대 인원 설정

### Phase 2 — 출시 후 2주

- **1/n 자동 정산**: 비용 등록 → 수락된 참여자 수로 균등 분할 자동 계산 → 납부 체크
- **카풀 그룹 매칭**: 드라이버가 그룹 생성, 동승 희망자가 신청, 드라이버가 수락

## 5. 채널

> 고객에게 어떻게 닿는가

- **카카오톡 링크 공유** (주채널): 주최자가 `join_code` URL을 단체방에 공유 → 자연스러운 바이럴
- **입소문**: 참여자가 다른 모임에서 주최자 역할을 맡으며 전파
- **구글 검색 SEO**: "모임 정산 앱", "이벤트 참여 신청 링크" 키워드 타겟
- **커뮤니티**: 수영·헬스 동호회 카페, 소모임 플랫폼(모임, 소모임) 홍보

## 8. 핵심지표

> 성공을 어떻게 측정하는가

### 성장 지표

| 지표                    | 목표 (3개월)                          | 측정 방법              |
| ----------------------- | ------------------------------------- | ---------------------- |
| 이벤트 생성 수          | 월 50건                               | `events` 테이블 COUNT  |
| 비로그인 참여 신청 비율 | 전체 신청의 40% 이상                  | `user_id IS NULL` 비율 |
| 주최자 재사용률         | 가입 후 2회 이상 이벤트 생성 비율 50% | 코호트 분석            |

### 건강 지표

| 지표                    | 목표                             | 측정 방법              |
| ----------------------- | -------------------------------- | ---------------------- |
| 정산 완료율             | 정산 등록 이벤트의 70% 납부 완료 | `is_paid = true` 비율  |
| DAU/MAU                 | 0.2 이상                         | 일별 활성 사용자       |
| 참여 신청 → 수락 전환율 | 80% 이상                         | `status accepted` 비율 |

### PMF 검증 기준 (성공 기준)

1. 주최자가 이벤트 생성 후 링크 하나로 참여 신청을 받는다
2. 비로그인 사용자도 공개 링크를 통해 신청 완료한다
3. 정산 1/n 계산과 납부 현황 추적이 자동화된다

## 9. 경쟁우위

> 쉽게 복사할 수 없는 강점

- **비로그인 참여 신청 UX**: 링크 클릭 후 이름만 입력하면 완료 → 기존 도구 대비 참여 마찰 최소화
- **주최자 중심 단일 인터페이스**: 공지·참여자·카풀·정산을 탭 하나로 전환 — 도구 전환 비용 제로
- **오픈 링크 기반 바이럴**: join_code URL이 카카오톡에 공유될 때마다 자동 홍보
- **소규모 모임 특화 UX**: 10~30명 규모에 맞춘 단순한 워크플로우 (대형 이벤트 플랫폼 대비 진입 장벽 낮음)
- **Supabase RLS 기반 보안**: 참여자별 데이터 접근 제어가 DB 레벨에서 강제됨 → 데이터 신뢰성
